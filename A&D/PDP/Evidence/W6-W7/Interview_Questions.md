 Stakeholder Interview Agenda

1. Present Current Development Progress
	•	Wireframes
	•	User Stories
	•	Backend Architecture
	•	Frontend Demo Pages
Goal: Gather feedback on usability, expectations, and potential missing elements

⸻

2. Present UI/UX Research: Leaderboard Concepts
	•	Show different versions of the leaderboard interface
	•	Present design directions: layout, animations, responsiveness
 Goal: Get stakeholder opinions on design, features, and ranking logic

⸻

3. Visual Design Feedback
	•	Let stakeholders compare and choose preferred designs
	•	Discuss functionality and user experience of each option
	•	Define a consistent design system for the application based on feedback

⸻

4. Further Questions

Visual Preferences
	•	What are your preferred color schemes?
	•	Should we include animations for user feedback/interactions?
	•	Do you have a logo you’d like us to use or should we design one?
	•	Any specific fonts we should consider?
	•	Are there any other design must-haves we haven’t mentioned?

 Technical & Functional Requirements
	•	Do you already have a hosting server for final deployment?
	•	Should user accounts for Venloop be predefined (whitelist-based) for high security, or should we implement an open sign-up system?

Admin Panel Usage
	•	Should the admin dashboard be PC-scaled or mobile-only?

⸻

5. Strategic & Process-Oriented Questions
	•	What is the real problem you hope this app will solve?
(Enhancing engagement? Inclusive team management? Simplified task flow?)
	•	How involved do you want to be in the development process?
	•	Daily / Weekly / Monthly updates?
	•	Only when new questions arise?
	•	Just review prototypes and final versions?

⸻

6. Open Feedback
	•	Any additional feedback on the current development progress or direction?
	•	Are there features or concerns we may have missed?
